\begin{abstract}
Generative models for molecules can produce structures that violate basic chemistry rules such as
bond valency limits, mass conservation, and charge balance. Rather than filtering out bad outputs
after the fact, this work proposes \textbf{ChemCS}, a system that enforces chemical constraints
during the generative process. We pair a small graph neural network (GNN) diffusion model
with a \textbf{supervisor loop} backed by Microsoft's Z3 theorem prover. At each reverse-diffusion 
step, the supervisor checks the candidate molecule against a set of chemical axioms and either 
commits the step, applies a lightweight correction, or backtracks. We also introduce a 
\textbf{constraint-aware training loss} with a curriculum schedule so the
model can learn to avoid violations on its own. Experiments across three reaction types show that
supervised generation achieves 100\% valency validity compared to roughly 15\% for
unsupervised generation with the same random-weight model, and that the training loss drops by
over 30\% within 100 epochs on a small dataset.
\end{abstract}

\section{Introduction}

Molecular generation is important for drug discovery, materials science, and chemical engineering.
Recent work has applied diffusion models --- originally designed for images --- to molecular graphs,
treating atoms as nodes and bonds as edges. The model starts from noise and iteratively denoises
toward a valid molecule.

The problem is that nothing in a standard diffusion model guarantees that the output will obey
chemistry. Carbon might end up with five bonds, mass might not be conserved across a reaction,
or the charges might not balance. The usual fix is to generate many candidates and throw away the
bad ones, which is wasteful and gives no guarantees.

ChemCS takes a different approach. Instead of post-hoc filtering, we insert a constraint checker
into the generation loop itself. The checker uses Z3, a satisfiability modulo theories (SMT) solver
from Microsoft Research, to formally verify whether each intermediate state satisfies a set of
chemical axioms. If it does not, the supervisor either fixes the state or rolls it back. The idea
is similar to how a human chemist would catch a mistake mid-calculation rather than waiting
until the end.

\section{Problem Formulation}

We represent a molecule as a set of atoms $\{a_1, a_2, \ldots, a_n\}$ where each atom $a_i$ has
an element type $e_i$, a bond count $b_i$, a formal charge $q_i$, and an implicit hydrogen count $h_i$.
The \textbf{adjacency matrix} $A \in \mathbb{R}^{n \times n}$ stores bond orders between atom pairs.

Given reactant molecules, we want to generate product molecules that satisfy:

\begin{enumerate}[nosep]
  \item \textbf{Bond valency:} For each atom, $b_i + h_i \leq V(e_i, q_i)$, where $V$ is the
        effective valency for element $e_i$ at charge $q_i$.
  \item \textbf{Mass conservation:} $\sum_{\text{reactants}} m_j = \sum_{\text{products}} m_k$
        within a tolerance $\epsilon$.
  \item \textbf{Charge conservation:} $\sum_{\text{reactants}} q_j = \sum_{\text{products}} q_k$.
  \item \textbf{Atom conservation:} For every element $e$, the count of $e$ atoms is the same
        on both sides.
\end{enumerate}

The generative model is a denoising diffusion process. Starting from a noisy version $x_T$ of the
reactant encoding, the model predicts the clean state $\hat{x}_0$ at each step $t$ and uses the
posterior to compute $x_{t-1}$. The supervisor wraps this loop.

\section{Methodology}

\subsection{Data Representation}

Each atom is encoded as a feature vector of length 12: a one-hot encoding over 10 supported
elements (H, C, N, O, F, P, S, Cl, Br, I), a normalized bond count, and a normalized formal charge.
The bond structure is an $n \times n$ adjacency matrix where entries are bond orders (0 -- 3).


\subsection{Diffusion Model}

The core model is a two-layer graph neural network (GNN). At each layer, every atom
updates its representation by combining its own features with the average of its
bonded neighbors' features:
\[
  h_i^{(l+1)} = \text{ReLU}\!\left(W_\text{self}^{(l)} h_i^{(l)}
    + W_\text{neigh}^{(l)} \frac{1}{|\mathcal{N}(i)|} \sum_{j \in \mathcal{N}(i)} h_j^{(l)}\right)
\]
\noindent where $h_i^{(l)}$ is the feature vector for atom $i$ at layer $l$,
$W_\text{self}^{(l)}$ and $W_\text{neigh}^{(l)}$ are learned weight matrices,
$\mathcal{N}(i)$ is the set of atoms bonded to atom $i$,
and $\text{ReLU}(x) = \max(0, x)$ clips negative values to zero.
After two such layers, a linear head predicts atom types and a bond head predicts
bond orders (0--3) for each pair of atoms.

The model supports two noise schedules that control how quickly the molecule moves
from clean to noisy (and back). The \textbf{linear} schedule adds noise at a steady,
fixed rate; the \textbf{cosine} schedule (Nichol \& Dhariwal, 2021) adds noise more
slowly at the start and end, giving the model more room to make fine-grained
adjustments when the molecule is nearly clean.

\subsection{Constraint Verification Layer}

The constraint checker:

\begin{itemize}[nosep]
  \item \textbf{Z3 backend:} Encodes mass conservation as Z3 real arithmetic, charge and atom counts
        as integer arithmetic, and valency as integer inequality queries. Each check is posed as
        ``is this constraint violated?''---if the solver returns SAT, the violation is real.
\end{itemize}

This checks four axioms (valency, mass, charge, atom conservation) and returns a
\texttt{ConstraintResult} with a boolean \texttt{sat} field and a list of violation strings.

\subsection{Supervisor Loop}

The supervisor wraps around the diffusion model's reverse process (Algorithm~1):

\begin{enumerate}[nosep]
  \item Encode the concatenated reactant atoms and apply full noise ($t = T$).
  \item For each step $t = T, T{-}1, \ldots, 1$:
    \begin{enumerate}[nosep]
      \item Run one reverse step to get a candidate $(x_{t-1}, A_{t-1})$.
      \item Decode the candidate into a \texttt{MolecularState}.
      \item Check valency constraints (mid-trajectory) or full conservation ($t = 1$).
      \item If valid: commit. If not: attempt a correction (fix valency, charge, mass). If the fix
            works, commit. Otherwise, re-sample up to \texttt{max\_retries} times.
      \item If all retries fail: backtrack to the previous committed state and re-try from $t+1$.
    \end{enumerate}
  \item After the loop, run a final full-reaction check and return the result with metrics.
\end{enumerate}

Correction strategies include reducing excess bonds on overvalenced atoms, adjusting implicit
hydrogens to match a target mass, and shifting formal charges on the lightest atoms to match a
target net charge.

\subsection{PyTorch Training Module}

A key insight from the midterm was that the supervisor ``silently repairs'' violations without
the model learning anything from its mistakes. To address this, we built a PyTorch training
module with the same GNN architecture:

```latex
\begin{itemize}[nosep]
  \item \textbf{Constraint-aware loss:} The training loss combines four terms: denoising MSE,
        bond cross-entropy, a differentiable valency penalty, and an element conservation penalty.
        In other words, the model is simultaneously scored on how accurately it predicts the
        molecule, the bond types, and whether it respects chemistry rules.
        The valency penalty uses soft probabilities over element types so gradients flow through:
        \[
          \mathcal{L}_\text{val} = \text{mean}\!\left(\text{ReLU}\!\left(\sum_j \mathbb{E}[\text{bond}_{ij}] - \sum_e p(e_i = e) \cdot V_e\right)\right)
        \]
        \noindent where $\mathbb{E}[\text{bond}_{ij}]$ is the expected number of bonds on atom $i$,
        $p(e_i = e)$ is the predicted probability that atom $i$ is element $e$,
        and $V_e$ is the maximum allowed bonds for element $e$ (its valency).
        The ReLU ensures only \emph{excess} bonds are penalized/ This means that if the atom is within its bond
        limit, the penalty is zero. Soft probabilities are used rather than hard element
        assignments so that gradients can flow back through the loss during training.

  \item \textbf{Curriculum schedule:} Training proceeds in two phases, similar to teaching
        someone a skill before adding constraints. During the first 25\% of training (warmup),
        constraint penalties are zeroed so the model can focus on learning basic denoising
        without competing objectives pulling it in different directions. After warmup, the
        penalties ramp linearly to their full weight, requiring the model to satisfy chemistry
        rules on top of what it already learned.
\end{itemize}

Trained weights are exported from PyTorch to the NumPy model via a \texttt{export\_to\_numpy()}
function, so the supervisor can use trained weights at inference without requiring PyTorch.

\subsection{Baselines}

We compare three configurations:

\begin{itemize}[nosep]
  \item \textbf{Unsupervised (raw):} Run the diffusion model forward and backward with no constraint
        checking. This is the standard generative approach.
  \item \textbf{Supervised (untrained):} Same random-weight model, but wrapped in the supervisor loop
        with correction and backtracking.
  \item \textbf{Supervised (trained):} The model is first trained on a small molecule dataset, then
        exported to NumPy and run with the supervisor.
\end{itemize}

\subsection{Evaluation Metrics}

\textbf{Valency validity} is the fraction of generated molecules where every atom's total bond count
(explicit + implicit H) does not exceed its effective valency. This is the primary metric.

\textbf{Full conservation validity} requires mass, charge, and atom counts to all be conserved.
This is harder and depends heavily on the model having learned the right element distributions.

\textbf{Commit rate} is the fraction of diffusion steps that passed constraints on the first try
(without correction or backtracking).

\section{Results}

\subsection{Constraint Verification}

The constraint checker correctly identifies all tested cases:

\begin{itemize}[nosep]
  \item Valid SN2 reaction (CH$_3$Br + OH$^-$ $\rightarrow$ CH$_3$OH + Br$^-$): all axioms satisfied.
  \item Valid combustion (CH$_4$ + 2O$_2$ $\rightarrow$ CO$_2$ + 2H$_2$O): all axioms satisfied.
  \item Overvalenced carbon (5 bonds): correctly flagged.
  \item Charge mismatch (neutral Br instead of Br$^-$): charge violation caught.
\end{itemize}

\subsection{Supervised vs Unsupervised Generation}

Figure~1 shows valency validity across 50 generation runs per reaction type with random (untrained)
model weights.

\begin{figure}[H]
  \centering
  \includegraphics[width=0.85\textwidth]{figures/fig2_valency_comparison.png}
  \caption{Supervised generation achieves 100\% valency validity across all three reaction types.
           Unsupervised generation averages roughly 15\%. Both use the same random-weight GNN.}

The supervisor brings valency validity from 12--22\% (unsupervised) to 100\% (supervised) by
correcting or backtracking at every step. The tradeoff is time: supervised runs take about 3--4$\times$
longer than unsupervised runs due to the constraint checking and re-sampling overhead.

Full conservation (mass + charge + atoms) remains at 0\% for both, which is expected since the model
has random weights and cannot predict correct element types. This motivates the training module.

\subsection{Training Performance}

We trained the PyTorch GNN on a 3-molecule dataset (water, methane, ethanol) for 200 epochs.
Figure~2 shows the loss curve.

\begin{figure}[H]
  \centering
  \includegraphics[width=0.85\textwidth]{figures/fig1_training_loss.png}
  \caption{Training loss over 200 epochs. The valency penalty starts high and drops sharply as the

The valency penalty drops from $\sim$5.5 down to below 0.2, showing the model is learning to respect
valency constraints through the differentiable penalty. The bond cross-entropy dominates the total
loss, which is expected given the discrete nature of bond prediction.

\subsection{Progress from Midterm}

At the midterm, we had the core constraint checker (mass, charge, valency with Z3 and pure-Python
backends) and the basic supervisor loop. The supervisor at that point improved valency validity from
about 8\% to 32\%.

Since then, we have:

\begin{itemize}[nosep]
  \item Added \textbf{atom/element conservation} as a fourth chemical axiom in both backends.
  \item Added a \textbf{charge correction} strategy to the supervisor.
  \item Built a \textbf{PyTorch training module} with a constraint-aware loss and curriculum schedule.
  \item Added \textbf{cosine noise schedule} support.
  \item Added \textbf{model save/load} for persisting trained weights.
  \item Implemented \textbf{weight export} from PyTorch to NumPy for inference.
  \item Expanded the benchmark to \textbf{3 reaction types} (SN2, combustion, acid-base) with 50 runs each.
  \item Added detailed \textbf{generation metrics} (commit rate, corrections, backtracks, timing).
  \item Grew the test suite from $\sim$60 tests to \textbf{118 tests}, all passing.
  \item Improved the supervisor's valency validity from 32\% to \textbf{100\%} by refining correction strategies.
\end{itemize}

\section{Discussion}

This work shows the value of integrating formal verification with generative models:

\begin{itemize}[nosep]
  \item \textbf{Constraint checking is cheap relative to generation.} Adding Z3-backed checks at
        each step only adds $\sim$3$\times$ overhead, which is acceptable for safety-critical applications.
  \item \textbf{Correction beats rejection.} Rather than filtering bad outputs, correcting them
        mid-trajectory is more efficient and always produces a result.
  \item \textbf{Training and enforcement are complementary.} The supervisor catches what the model
        gets wrong; the constraint-aware loss helps the model get fewer things wrong over time.
\end{itemize}

The main tension is between the supervisor silently fixing violations (which hides errors from
the user) and the model needing to see its own errors to learn. The curriculum schedule is our
solution: early in training, the model makes and sees mistakes freely; later, the penalties
force it to internalize the constraints.

\section{Limitations and Future Work}

Limitations include:

\begin{itemize}[nosep]
  \item The training dataset is very small (3 molecules). Real chemistry would require
        parsing SMILES strings from databases like ZINC or QM9 via RDKit.
  \item Full conservation validity is still 0\% with random weights. Training on real data is
        needed to achieve meaningful conservation.
  \item The bond prediction is pairwise ($O(n^2)$), which does not scale to large molecules.
  \item The model lacks ring detection, stereochemistry, and aromaticity.
\end{itemize}

Future work:

\begin{itemize}[nosep]
  \item Parse real molecular datasets with RDKit and train on thousands of molecules.
  \item Benchmark 1,000 generated reactions end-to-end.
  \item Add ring constraints and 3D coordinate generation.
  \item Integrate the supervisor into the training loop directly (not just via loss penalties).
  \item Explore more sophisticated correction strategies (e.g., using the solver to suggest fixes).
\end{itemize}

\section{Conclusion}

ChemCS demonstrates that constraint satisfaction and generative models can be combined in a practical
way. The supervisor loop enforces chemical validity at every step of the diffusion process, bringing
valency validity from $\sim$15\% to 100\% even with an untrained model. The constraint-aware training
loss with curriculum scheduling provides a path for the model to learn the constraints itself, reducing
the supervisor's burden over time.

The key takeaway is the difference between \emph{enforcing} correctness at inference time and
\emph{teaching} correctness during training. Both are necessary: enforcement gives guarantees, and
teaching gives efficiency. ChemCS provides both.

The system is fully tested (118 passing tests), supports both Z3 and pure-Python constraint checking,
and is designed so that a trained PyTorch model can be exported to a lightweight NumPy model for
deployment. The code, tests, and demo are available in the project repository.
